🚀 Pendahuluan

TechnoPiousQuiz adalah aplikasi web untuk menguji kemampuan pemrograman peserta melalui kuis interaktif. Aplikasi ini dibangun menggunakan HTML, CSS, dan JavaScript murni (Vanilla JS) tanpa framework eksternal.

Fitur Utama Aplikasi

FiturDeskripsi
Mode ManualPengawas mencatat jawaban secara manual dengan timer
Mode TesSesi ujian otomatis dengan soal bergilir
Bank Soal1000+ sintaks dari 7 bahasa pemrograman
Timer GandaTimer per soal dan timer sesi keseluruhan
ValidasiPenilaian benar/salah oleh pengawas
EksporCSV, Spreadsheet, JSON, Clipboard
ResponsiveTampilan optimal di desktop dan mobile
ℹ️ Konsep Dasar Aplikasi ini menggunakan paradigma Single Page Application (SPA) sederhana di mana semua konten berada dalam satu file HTML dan navigasi antar tab dilakukan dengan menyembunyikan/menampilkan elemen (tidak ada reload halaman).

Alur Kerja Aplikasi

  1. Pengguna mengatur nama peserta melalui modal dialog
  2. Memilih tab: Manual, Tes, atau Hasil
  3. Pada Mode Tes: konfigurasi sesi → mulai tes → soal tampil → peserta menjawab
  4. Setelah sesi selesai, pengawas memvalidasi jawaban di tab Hasil
  5. Data dapat diekspor ke berbagai format

🏗️ Struktur HTML

Aplikasi memiliki struktur HTML yang terorganisir dalam beberapa bagian utama:

Kerangka Dasar

<!DOCTYPE html>
<html lang="id">
<head>
  // Meta tags, title, dan CSS styles
</head>
<body>
  // Header dengan logo dan jam
  // Container utama

  // Tiga tab konten: manual, test, results
  // Modal dialog
  // JavaScript di akhir body
</body>
</html>

Komponen Utama HTML

  • Header: Logo, badge peserta, jam realtime
  • Quick Stats: 4 kartu statistik (total soal, benar, akurasi, rata-rata waktu)
  • Tab Navigation: 3 tombol tab (Manual, Tes, Hasil)
  • Tab Manual: Timer + input jawaban + tabel hasil
  • Tab Tes: Konfigurasi + area tes + control bar
  • Tab Hasil: Grafik + log jawaban + tombol ekspor
  • Modal: Dialog untuk edit nama peserta
  • Overlay Pause: Tampilan saat tes dijeda
💡 Tips Perhatikan penggunaan atribut id pada setiap elemen penting. ID ini digunakan JavaScript untuk mengakses dan memanipulasi elemen menggunakan document.getElementById().

🎨 CSS Variables (Custom Properties)

Aplikasi menggunakan CSS Variables (didefinisikan di :root) untuk mengelola tema gelap secara konsisten. Ini memudahkan perubahan warna di seluruh aplikasi.

Definisi Variabel

:root {
  /* Warna latar */
  --bg: #060b14;
  --surface: #0d1525;
  --surface2: #131d33;

  /* Warna teks */
  --text: #e8ecf1;
  --text2: #8899b4;

  /* Warna aksen */
  --accent: #4f8cf7;
  --success: #10b981;
  --danger: #ef4444;
  --warning: #f59e0b;
  --purple: #8b5cf6;
}

Cara Penggunaan Variabel

.card {
  background: var(--surface);
  color: var(--text);
  border: 1px solid var(--border);
}
ℹ️ Keuntungan CSS Variables
  • Perubahan warna cukup di satu tempat
  • Mendukung tema gelap/terang dengan mudah
  • Nilai dapat diubah via JavaScript
  • Didukung semua browser modern (96%+)

Generate 1000 Soal

function generateFullBank() {
  fullSyntaxBank = [];
  for (const [lang, keywords] of Object.entries(languageKeywords)) {
    keywords.forEach(kw => {
      // Tentukan tingkat kesulitan
      let diff = 'basic';
      // ... logika penentuan difficulty
      fullSyntaxBank.push({ lang, keyword: kw, difficulty: diff });
    });
  }
  // Duplikasi hingga 1000 item
  while (fullSyntaxBank.length < 1000) { ... }
}
ℹ️ Setiap item soal memiliki struktur: { lang: "Python", keyword: "def", difficulty: "basic" }

⏱️ Logika Timer

Timer adalah fitur paling kritis. Aplikasi menggunakan setInterval untuk menghitung mundur.

Dual Timer System

function startTestTimers() {
  // Timer per soal (setiap detik)
  testPerSyntaxTimer = setInterval(() => {
    if (testPaused || !testSessionActive) return;
    testPerSyntaxRemaining--;
    updateTestTimerDisplay();
    if (testPerSyntaxRemaining <= 0) {
      submitCurrentAnswer(true);
      nextSyntax();
    }
  }, 1000);

  // Timer sesi (setiap detik)
  testSessionTimer = setInterval(() => {
    if (testPaused || !testSessionActive) return;
    testSessionRemaining--;
    updateTestSessionTimerDisplay();
    if (testSessionRemaining <= 0) {
      endTestSession('timeout');
    }
  }, 1000);
}

Menghentikan Timer

// Saat pause atau sesi berakhir
clearInterval(testPerSyntaxTimer);
clearInterval(testSessionTimer);
testPerSyntaxTimer = null;
testSessionTimer = null;
⚠️ Penting! Selalu bersihkan interval dengan clearInterval() untuk mencegah memory leak dan perilaku tak terduga.

📤 Ekspor Data

Aplikasi mendukung 4 format ekspor.

Metode Ekspor

function exportData(format) {
  switch(format) {
    case 'clipboard':
      navigator.clipboard.writeText(text);
      break;
    case 'csv':
    case 'spreadsheet':
    case 'json':
      downloadFile(filename, mimeType, content);
      break;
  }
}

Download File

function downloadFile(filename, mimeType, content) {
  const blob = new Blob([content], { type: mimeType });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url);
}

💡 Best Practices dalam Kode

1. Single Responsibility

Setiap fungsi memiliki satu tugas spesifik. Contoh: updateTestTimerDisplay() hanya mengupdate tampilan timer, tidak melakukan logika lain.

2. Penamaan yang Jelas

Variabel dan fungsi menggunakan nama deskriptif: testPerSyntaxRemaining, submitCurrentAnswer().

3. Komentar Kode

// ==================== MODE TES ====================
// Bagian ini menangani semua logika sesi ujian

4. Error Prevention

  • Validasi input sebelum memulai tes
  • Pengecekan testSessionActive sebelum operasi
  • Try-catch untuk operasi yang mungkin gagal

5. Cleanup

Timer selalu dibersihkan saat komponen tidak lagi digunakan.
